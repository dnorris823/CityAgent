<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 2: In-game verification of memory file explorer</name>
  <what-built>
Complete memory file explorer: tab navigation (Advisor/Memory), file list with core badges
and metadata, file content viewer, edit mode with Save Changes/Discard Changes, delete confirmation
with inline destructive sub-header, and core file protection (no Delete button for core files).
  </what-built>
  <how-to-verify>
1. Launch CS2 and load a city that has memory files (any city you have previously chatted with)
2. Open the CityAgent panel — verify the header shows "Advisor" and "Memory" tab buttons
   instead of "CityAgent AI Advisor" title text
3. Click the "Memory" tab — verify:
   a. A scrollable list of .md files appears
   b. Core files (_index.md, characters.md, etc.) appear first with a [core] badge
   c. Each row shows filename, size (e.g. "4.2KB"), and relative time (e.g. "2d ago")
4. Click a core file (e.g. _index.md) — verify:
   a. Sub-header shows: `← _index.md  [Edit] [core]` (no Delete button)
   b. File content appears in monospace font below the sub-header
5. Click Edit — verify:
   a. Content area switches to a textarea pre-populated with the file content
   b. Sub-header changes to: `← _index.md  [Save Changes] [Discard Changes]`
6. Make a small edit, click Save Changes — verify:
   a. Returns to view mode showing the updated content
   b. Switch to Advisor tab, then back to Memory tab, re-open the file — change persisted
7. Click a non-core file — verify:
   a. Sub-header shows: `← filename.md  [Edit] [Delete]`
   b. Click Delete — sub-header turns red-tinted: "Delete filename.md?  [Yes] [Discard Changes]"
   c. Click Discard Changes — returns to normal file view
8. Click the Advisor tab — verify:
   a. Chat interface appears as normal (messages, input bar, etc.)
   b. Click Memory tab again — verify it starts fresh at the file list (not the previous file view)
9. Verify drag behavior: click and drag on the header area BETWEEN the tabs and the close button — panel should drag. Clicking a tab button should NOT start a drag.
  </how-to-verify>
  <resume-signal>Type "approved" if all checks pass, or describe any issues</resume-signal>
