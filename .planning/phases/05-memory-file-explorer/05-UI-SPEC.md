# Phase 5 — UI Design Contract: Memory File Explorer

> Visual and interaction contract for Phase 5. Generated by gsd-ui-researcher.
> Consumed by gsd-planner, gsd-executor, and gsd-ui-auditor.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none — hand-authored CSS + React |
| Icon library | Twemoji CDN (SVG via `ca-emoji` background-image pattern) |
| Font | inherited (CS2 game font via `font-family: inherit`) |

**Note (from CLAUDE.md):** No shadcn, no Radix, no external component libs. CS2's embedded
browser (Coherent GT) is an older Chromium build. All styles in `UI/src/style.css` with `ca-`
BEM prefix. No CSS Grid. No `gap`. No `::placeholder`. No `:disabled` pseudo-class
(use `[disabled]` attribute selector). No `Array.at()`. Scrollbars via `-webkit-scrollbar`.

## Spacing Scale

All spacing expressed in `em` units (scaling with `fontSize` binding) using this reference
base of 16px. Pixels shown are the reference values at default 14px font size.

| Token | em value | Reference px | Usage |
|-------|----------|--------------|-------|
| xs | 0.25em | ~4px | Icon inline padding, tight row gaps (flexbox margin) |
| sm | 0.5em | ~8px | Compact element spacing, button padding vertical |
| md | 0.7em | ~10px | Default cell padding (matches existing `.ca-input-area` padding) |
| lg | 0.85em | ~12px | Content area padding (matches `.ca-messages` padding) |
| xl | 1em | ~14px | Section internal spacing |
| tab-height | 2.2em | ~31px | Tab bar minimum height (touch/click target) |
| sub-header-height | 2.4em | ~34px | Sub-header bar minimum height |

**Exceptions:** File list rows use 44px minimum touch-target height expressed as
`min-height: 2.8em` at 16px reference. This ensures legibility and click accuracy inside
the game panel at all font sizes.

**Coherent GT note:** All spacing uses `margin-*` properties on flex children — never `gap`.

## Typography

All type is `em`-relative to the panel's `fontSize` binding (default 14px, user-configurable).

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 0.92em (~13px at 14px base) | 400 (regular) | 1.55 | File content view (`<pre>`), textarea |
| Label | 0.85em (~12px) | 400 (regular) | 1.4 | File list metadata (size, relative time), sub-header secondary text |
| UI Text | 1em | 600 (semibold) | 1.3 | Tab labels, sub-header filename, section headings |
| Micro | 0.78em (~11px) | 400 (regular) | 1.2 | Lock label / core badge, error pill text |

**Weights in use:** 400 and 600 only — matches existing panel convention.

**Monospace:** File content (`<pre>`) uses `font-family: "Consolas", "Courier New", monospace`
at body size — identical to `.ca-markdown pre code` pattern in existing stylesheet.

## Color

All colors in `rgba()` — no hex shorthand. Inherits the existing panel dark translucent palette.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `rgba(12, 22, 36, 0.92)` | Panel background — already set by `.ca-panel` |
| Secondary (30%) | `rgba(22, 38, 58, 0.9)` | Tab bar background, sub-header background, file list row hover; matches `.ca-bubble--assistant` |
| Accent (10%) | `rgba(74, 158, 222, ...)` | See accent-reserved list below |
| Destructive | `rgba(180, 40, 40, 0.85)` | Delete confirmation sub-header tint, delete error notice |

**Accent reserved for:**
- Active tab indicator (bottom border on active tab: `rgba(74, 158, 222, 0.8)`, 2px)
- Active tab label text (`#4a9ede`)
- Inline link text in file content (inherits `.ca-markdown a` rule)
- Edit button hover state (border-color transition, matches `.ca-btn-icon:hover`)
- Focus ring on textarea edit area (matches `.ca-input:focus` pattern)
- Back arrow `←` text color

**Inactive tab:** `color: rgba(160, 180, 200, 0.7)`, no bottom border, `background: transparent`.

**Destructive accent reserved for:**
- Delete button idle: same as `.ca-btn-icon` (no destructive tint until confirmation)
- Delete confirmation sub-header: `background: rgba(140, 20, 20, 0.35)`, border `rgba(180, 40, 40, 0.4)`
- Delete error inline notice: `color: rgba(255, 100, 100, 0.9)`, background `rgba(140, 20, 20, 0.3)`
- Save error notice above textarea: same destructive palette

**Lock / core badge:** `color: rgba(160, 180, 200, 0.55)` — visually subdued, not accent.

### Content Area Loading State

When `readMemoryFile` trigger has been fired but `memoryOpResult` is still `""` (loading),
show `.ca-mem-content--loading` with text "Loading..." in place of file content.
No animated dots needed — this is a local file read (sub-100ms).

### Error Display

**Save error:** `.ca-mem-error` banner appears between sub-header and textarea.
Copy: "Save failed — [error message from memoryOpResult]". Edit mode persists.

**Delete error:** `.ca-mem-subheader__error` inline text appended in sub-header.
Copy: "Delete failed — [error message]". File view persists.

## Lock Icon

**Decision (Claude's Discretion):** Use a text badge `[core]` styled as a small pill label
rather than an emoji. Rationale: Coherent GT's emoji font support is unreliable; the Twemoji
CDN approach (background-image on `.ca-emoji` span) requires knowing the exact Unicode
codepoint and building the CDN URL — fragile for a lock icon that is purely decorative.
A text badge is guaranteed to render and is instantly legible.

```css
.ca-mem-badge--core {
  font-size: 0.72em;
  padding: 0.1em 0.35em;
  border-radius: 3px;
  background: rgba(60, 100, 150, 0.2);
  border: 1px solid rgba(60, 100, 150, 0.3);
  color: rgba(160, 180, 200, 0.6);
  white-space: nowrap;
  margin-right: 0.4em;
  flex-shrink: 0;
}
```

Placed as the first child inside `.ca-mem-list__icon` position (leftmost in row, before filename).

## Relative Time Formatting

Implemented as a pure utility function `formatRelativeTime(unixSeconds: number): string`.
No external library. Uses `Date.now()` arithmetic.

| Age | Format |
|-----|--------|
| < 60 seconds | `just now` |
| 1–59 minutes | `Xm ago` |
| 1–23 hours | `Xh ago` |
| 1–6 days | `Xd ago` |
| 7+ days | `Xw ago` (weeks) |

Cap at weeks — no months/years needed given memory file lifecycle.

---

## File Metadata Display Format

Each file row right-aligned meta: `{size}  {relative time}`

Size formatting: `{N.N}KB` for files >= 1KB (one decimal place), `{N}B` for files < 1KB.
Source: `size_kb` field from `memoryFilesJson` (already in KB from C#).

Example row layout:
```
[core] _index.md              4.2KB  2d ago
       lore.md                1.8KB  just now
       districts.md           3.1KB  5h ago
```

## `memoryOpResult` Reset Strategy (Claude's Discretion)

React resets its interpretation of `memoryOpResult` when a new trigger fires:
- On `readMemoryFile` trigger: set local state `fileContent = ""` (loading); bind to
  next `memoryOpResult` value.
- On `writeMemoryFile` trigger: set local state `saveResult = null`; consume next value.
- On `deleteMemoryFile` trigger: set local state `deleteResult = null`; consume next value.

C# resets `memoryOpResult` to `""` at the start of each new trigger handler (before
the operation). This means React can detect completion by checking if `memoryOpResult`
transitioned from `""` to a non-empty value after the trigger fired.

## Drag Behavior with Tab Bar

The tab bar is placed **inside** `.ca-panel__header`, which carries the `onMouseDown` drag
handler. Tab buttons must call `e.stopPropagation()` on their `mousedown` event to prevent
drag from activating on tab clicks. The `.ca-panel__header-actions` div already does this
via `stopDragPropagation` — tab buttons follow the same pattern.

Tab bar is **not** the drag handle. The drag-active zone is any part of `.ca-panel__header`
that is NOT a tab button or action button.

---

## `writeMemoryFile` Thread Strategy (Claude's Discretion)

Write synchronously on the main thread for Phase 5. The files are small markdown documents
(< 50KB). Synchronous write from the trigger handler keeps the implementation simple and
consistent with the existing `NarrativeMemorySystem.WriteFile()` call pattern.
If write latency becomes noticeable in practice, migrate to async in a follow-up.
