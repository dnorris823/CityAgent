# Phase 2 — UI Design Contract: Chat UI Polish

> Visual and interaction contract for the chat panel polish phase.
> Generated by gsd-ui-researcher. Consumed by gsd-planner, gsd-executor, gsd-ui-auditor.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none — hand-rolled React + CSS |
| Icon library | none — Unicode glyphs inline; emoji via Twemoji SVG CDN |
| Font | inherited from CS2 game UI (Coherent GT) |

**Rationale:** This is a CS2 mod UI running in Coherent GT (older Chromium). No external libraries can be bundled. Shadcn is not applicable. All components are hand-authored in `UI/src/components/CityAgentPanel.tsx` and styled in `UI/src/style.css`.

**Coherent GT constraints (hard — cannot deviate):**
- No CSS `gap` property
- No `::placeholder` pseudo-element
- No `:disabled` pseudo-class (use `[disabled]` attribute selector)
- No CSS Grid (use flexbox exclusively)
- No `Array.at()`, Unicode property escapes (`\p{}`), or other modern JS features in utilities
- Emoji must use Twemoji SVG background-image spans via `.ca-emoji` class

## Spacing Scale

Source: existing `style.css` — `em`-based scaling from `fontSize` binding (default 14px).

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (0.28em @ 14px) | Dot margins in loading animation, inline chip spacing |
| sm | 8px (0.55em @ 14px) | Bubble padding vertical, input area padding, chip bottom margin |
| md | 12px (0.85em @ 14px) | Bubble padding horizontal, messages area padding |
| lg | 16px (1.1em @ 14px) | Input row element spacing |
| xl | 24px | Not used directly — composed from em units |

**Key rule:** All padding and margins in new components use `em` units (not `px`) so they scale with the `fontSize` binding. Inline `style` props are used only for dynamic computed values (position, pixel dimensions from bindings). Static layout uses `ca-*` CSS classes.

**New spacing tokens for this phase:**

| Element | Value | Notes |
|---------|-------|-------|
| System pill padding | `0.2em 0.75em` | Matches existing chip height rhythm |
| System pill margin | `0.3em 0` | Fits between adjacent bubbles without crowding |
| Loading status text top margin | `0.4em` | Separates text from dot row |
| Welcome block padding | `1.5em 1em` | Generous — fills empty panel center |
| Queued chip: matches `.ca-screenshot-chip` | `0.2em 0.7em` | Exact match to existing chip token |
| Code block language label padding | `0.15em 0.5em` | Small label above `<pre>` |

## Copywriting Contract

Source: CONTEXT.md D-06, D-11; Claude's discretion area.

### Loading status phrases (rotate every 2.5 seconds, loop)

Exactly 6 phrases. Tone: immersive narrator, CityPlannerPlays vibe.

1. "Surveying the city..."
2. "Consulting the records..."
3. "Studying the districts..."
4. "Reviewing the latest reports..."
5. "Checking in with the planners..."
6. "Analyzing the situation..."

Implementation: `phrases[Math.floor(Date.now() / 2500) % phrases.length]` recalculated on a `setInterval` of 2500ms while `isLoading` is true. Reset to phrase index 0 each time loading begins.

### Welcome greeting variants (randomly selected, reselected on New Chat)

Exactly 5 variants. Selection: `phrases[Math.floor(Math.random() * phrases.length)]` on mount and on `clearChat` trigger.

1. "Welcome back, Mayor. The city awaits."
2. "Your advisor is ready. What would you like to know?"
3. "The city has been busy. Ask me anything."
4. "Ready to help. What's on your mind, Mayor?"
5. "All systems nominal. How can I assist?"

## Interaction Contracts

### System notice pill (D-01 through D-05)

- Rendered in place within the message list, ordered by insertion time
- Cleared by "New Chat" exactly like regular messages
- Never sent to the Claude API (C# filters `role == "system"` before payload construction)
- `ChatMessage` interface gains `role: "user" | "assistant" | "system"` — React switch renders `.ca-notice-pill` for `"system"` role
- No close/dismiss button on notice pills (persist until New Chat)
- Notice type (error vs. warning) communicated by a subtype field or by matching `[Error]:` prefix vs. `[Warning]:` / `[RateLimit]:` prefix patterns

### Loading state (D-06 through D-08)

- Loading bubble: existing `.ca-bubble.ca-bubble--assistant` wrapper containing `.ca-loading-dots` + `.ca-loading-status` text below
- Status text cycles independently via `setInterval`; interval is cleared in `useEffect` cleanup when `isLoading` becomes false
- Input field: remains enabled while loading (not grayed, not `[disabled]`)
- Send button: behavior depends on queued message state (see below)

### Type-ahead / queued message (D-08 through D-10)

- When `isLoading === true` and user presses Enter or clicks Send:
  - Message is NOT dispatched via `sendMessage` trigger
  - Input clears
  - Queued message stored in local React state (`queuedMessage: string | null`)
  - `.ca-queued-chip` appears above the input row showing `queued: {truncated text}` with ✕ dismiss
- Send button state while a message is queued: `[disabled]` + `opacity: 0.35` (same `.ca-send-btn[disabled]` rule)
- ✕ on queued chip: sets `queuedMessage` to null, chip disappears
- When `isLoading` transitions from `true` to `false` (detected in `useEffect` watching `isLoading`):
  - If `queuedMessage` is non-null: call `safeTrigger("cityAgent", "sendMessage", queuedMessage)` then set `queuedMessage` to null
  - Chip disappears simultaneously with the send (state clears in the same handler)
- C# side: `m_QueuedMessage` implementation detail is left to planner/executor (noted in CONTEXT.md as Claude's discretion)

### Welcome state (D-11 through D-13)

- Welcome block visible when: `messages.length === 0 && !isLoading`
- Disappears: instantly when first message pushed to history (no CSS transition, no fade)
- Reappears: after `clearChat` trigger clears message history back to 0
- No separate greeting for "New Chat" vs. first open — same pool

### Markdown fixes (D-14 through D-20)

These are renderer behavior contracts, not visual contracts. Each fix is scoped to `renderMarkdown.ts`.

| Fix | Requirement |
|-----|-------------|
| Nested lists | A list item followed by an indented list line (`  - `) must produce `<li>...<ul><li>...</li></ul></li>` not two flat `<li>` elements |
| Italic regex | Replace `(?<!\w)_(.+?)_(?!\w)` with a Coherent GT–safe equivalent: use a character class boundary check instead of lookbehind (e.g. require preceding char is whitespace/start or punctuation) |
| Bold + heading coexistence | Verify `## **Header**` and `**bold** in paragraph` in same response produce correct output — fix any regex ordering issue found |
| Code block language label | When `fenceMatch[1]` is non-empty, emit `<span class="ca-code-lang">{lang}</span>` immediately before `<pre>` |
| `<thinking>` stripping | Strip `<thinking>...</thinking>` blocks (including content) before display. Preferred location: C# in `CityAgentUISystem` before writing to `m_History`. Fallback: strip in `renderMarkdown.ts` as first pass if C# approach is out of scope for this phase. |
| Malformed markdown | Best-effort — unmatched markers render as literal text. No change from current behavior. |
| User messages | Plain text only — no `renderMarkdown` call. Current behavior unchanged. |

## Implementation Notes for Executor

1. **`ChatMessage` interface** — add `"system"` to the `role` union type. C# `ChatMessage` class gains `role = "system"` as a valid value.

2. **Message renderer switch** — the current `ca-bubble--${msg.role}` pattern must become a conditional:
   - `role === "user"` → existing user bubble
   - `role === "assistant"` → existing assistant bubble
   - `role === "system"` → new `.ca-notice-pill` (detect error vs. warning by checking if content starts with `[Error]` → error variant; otherwise warning variant)

3. **Loading status text interval** — use `useRef` for the interval ID, start in `useEffect` when `isLoading` becomes true, clear on cleanup. Do not use `useState` for the phrase index (causes extra re-renders); derive current phrase from `Date.now()` instead.

4. **Queued message state** — add `const [queuedMessage, setQueuedMessage] = useState<string | null>(null)` to `CityAgentInner`. Update `handleSend` to branch on `isLoading`. Add `useEffect` watching `isLoading` for the auto-send drain.

5. **`ca-welcome` rendering** — render as a sibling to the `messages.map(...)` output inside `.ca-messages`, conditionally: `{messages.length === 0 && !isLoading && <div className="ca-welcome">{selectedGreeting}</div>}`. Greeting selected once via `useState` initializer; re-selected by a `useEffect` that watches the `clearChat` trigger (or simply re-selected when `messages.length` transitions to 0).

6. **`renderMarkdown.ts` nested lists** — requires a two-pass approach or recursive indent tracking. The current flat loop collects any line matching `/^[\s]*[-*+]\s/` into a single `<ul>` without tracking indent depth. Fix: collect lines and track indent level to build nested `<ul>` elements.

7. **Coherent GT italic regex fix** — replace the lookbehind `(?<!\w)_(.+?)_(?!\w)` at `renderMarkdown.ts:206` with a safe alternative. Suggested pattern: `/(^|[\s.,!?;:'"([{])_([^_]+?)_(?=[\s.,!?;:'")\]}]|$)/g` applied to the full line with a non-capturing context restore.
