    ```csharp

    {
        if (!m_NarrativeMemory.IsInitialized || m_History.Count == 0) return;

        try
        {
            var messages = m_History.Select(m => (m.role, m.content, m.hadImage));
            string markdown = NarrativeMemorySystem.ChatHistoryToMarkdown(
                messages, m_NarrativeMemory.SessionNumber, m_NarrativeMemory.CityName);
            _ = m_NarrativeMemory.SaveChatSessionAsync(markdown);  // fire-and-forget per D-12

        }
    }
    ```
    Key change: `m_NarrativeMemory.SaveChatSession(markdown)` becomes `_ = m_NarrativeMemory.SaveChatSessionAsync(markdown)`. The ChatHistoryToMarkdown call (pure function, no I/O) stays synchronous on the main thread. Only the file write is fire-and-forget.

    **4. Move screenshot base64 encoding to background thread (D-13, CORE-01):**

    Replace the current synchronous screenshot read in OnUpdate (the `File.Exists(m_ScreenshotPath)` block, lines ~129-144):

    ```csharp
    else if (File.Exists(m_ScreenshotPath))
    {
        m_ScreenshotWaitFrames = -1;  // Stop polling immediately
        string path = m_ScreenshotPath;
        _ = Task.Run(() =>
        {
            try
            {
                byte[] png = File.ReadAllBytes(path);
                File.Delete(path);
                string base64 = Convert.ToBase64String(png);
                // Marshal back to main thread state
                m_PendingBase64Image = base64;
                m_HasScreenshot.Update(true);
                Mod.Log.Info($"Screenshot loaded: {png.Length} bytes.");
            }
            catch (Exception ex)
            {
                Mod.Log.Error($"Screenshot read failed: {ex.Message}");
            }
        });
    }
    ```

    Note: `m_HasScreenshot.Update(true)` is a ValueBinding update — CS2's binding system is designed to handle cross-thread updates (it queues the update for the next UI frame). The `m_PendingBase64Image` field is only read when `OnSendMessage` fires (user action), which won't race with the background write in practice. This matches the existing fire-and-forget pattern used throughout the codebase.
