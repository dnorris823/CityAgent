**CityToolRegistry.cs changes (per D-11, D-21):**

1. Add a private `IsToolEnabled` method that maps tool names to setting toggles. Memory tools always return true (per D-10):
```csharp
private static bool IsToolEnabled(string toolName)
{
    var s = Mod.ActiveSetting;
    if (s == null) return true; // safety: if settings not loaded, include all

    return toolName switch
    {
        "get_population"       => s.EnablePopulationTool,
        "get_building_demand"  => s.EnableBuildingDemandTool,
        "get_workforce"        => s.EnableWorkforceTool,
        "get_zoning_summary"   => s.EnableZoningSummaryTool,
        "get_budget"           => s.EnableBudgetTool,
        "get_traffic_summary"  => s.EnableTrafficSummaryTool,
        "get_services_summary" => s.EnableServicesSummaryTool,
        _                      => true  // memory tools and any future non-data tools
    };
}
```

2. Update `GetToolsJson()` to filter: add `if (!IsToolEnabled(tool.Name)) continue;` as the first line inside the `foreach (var tool in m_Tools.Values)` loop.

3. Update `GetToolsJsonOpenAI()` with the same filter: add `if (!IsToolEnabled(tool.Name)) continue;` as the first line inside the `foreach` loop.

Note: `Dispatch()` does NOT filter -- if Claude somehow calls a disabled tool (shouldn't happen since it's not in the schema), it still works. The filter is at the serialization boundary only.
